componentsSeparatedByString:withOpeningQuote:closingQuote:singleQuote:

Abstract: Get an array of string componets. 
- (NSArray *)componentsSeparatedByString:(NSString *)separator withOpeningQuote:(NSString *)openingQuote closingQuote:(NSString *)closingQuote singleQuote:(NSString *)singleQuote includeEmptyComponents:(BOOL)flag;

Returns an NSArray containing substrings from the receiver that have been divided by separator. The substrings in the array appear in the order they did in the receiver.

Parameters

NameDescription
separatorThe delimeter used to seperate the strings.
openingQuoteThe string used to begin a quoted component, a component where the delimiter is ignored.
closingQuoteThe string used to end a quoted component, a component where the delimiter is ignored.
singleQuoteThe string used to quote a single character.
flagSet to true if empty string are wanted.
Result: A NSArray of components.

containsAnyCharacterFromSet:options:

Abstract: Test if a string contains a character from a set. 
- (BOOL)containsAnyCharacterFromSet:(NSCharacterSet *)set options:(unsigned int)mask;

Returns YES if the receiver contains any character within set within the range range, otherwise returns NO. The following options may be specified in mask by combining them with the C bitwise OR operator:

Search Option Effect
NSCaseInsensitiveSearch Ignores case distinctions among characters.
NSLiteralSearch Performs a byte-for-byte comparison. Differing literal sequences (such as composed character sequences) that would otherwise be considered equivalent are considered not to match. Using this option can speed some operations dramatically.
NSBackwardsSearch Performs searching from the end of the range toward the beginning.
See "Strings" for details on these options. Raises an NSInvalidArgumentException if any of the arguments are nil.

Parameters

NameDescription
setThe set of characters to look for.
maskMask values determining how to search.
Result: Returns YES if the receiver contains any of the characters.

containsString:options:range:

Abstract: Test if a string contains a substring. 
- (BOOL)containsString:(NSString *)subString options:(unsigned)mask range:(NSRange)range;

Returns YES if subString occurs within range in the receiver. If subString isn't found, returns a NO. The following options may be specified in mask by combining them with the C bitwise OR operator:

Search OptionEffect
NSCaseInsensitiveSearchIgnores case distinctions among characters.
NSLiteralSearchPerforms a byte-for-byte comparison. Differing literal sequences (such as composed character sequences) that would otherwise be considered equivalent are considered not to match. Using this option can speed some operations dramatically.
NSBackwardsSearchPerforms searching from the end of the range toward the beginning.
NSAnchoredSearchPerforms searching only on characters at the beginning or end of the range. No match at the beginning or end means nothing is found, even if a matching sequence of characters occurs elsewhere in the string.


Raises an NSRangeException if any part of range lies beyond the end of the string.

Parameters

NameDescription
subStringThe string to search for.
maskMask values determining how to search.
rangeThe range within the string to search.
Result: Returns YES if substring found otherwise NO.

nonLossyASCIIString

Abstract: Get a 7 bit ASCII representation of a NSString containing Unicode characters. 
- (const char *)nonLossyASCIIString;

Returns a c string using NSNonLossyASCIIStringEncoding encoding, a 7-bit verbose ASCII to represent all Unicode characters. The returned C string will be automatically freed just as a returned object would be released; your code should copy the C string, if it needs to store the C string outside of the autorelease context in which the C string is created.

Result: A c string with NSNonLossyASCIIStringEncoding encoding.
